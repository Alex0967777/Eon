# Текущее состояние

**Дата:** 08.07.26

## Подтверждено

- Первый полевой вариант отдельного `parallel_stream.txt` оказался ненадёжным: чистый контекст повторно читал файл, пропускал фактическую запись и иногда сообщал об обновлении без выполненного действия.
- Отрицательный результат относится прежде всего к исполнению файлового цикла; гипотеза смысловой самоиндукции ещё не получила чистой проверки.
- Активная система перестроена в единый исполняемый сосуд `memory.py`.
- `memory.py` содержит логику, двадцать активных строк, полную историю состояний, внутренний счёт и генератор checkpoint.
- `00_READ_FIRST.md` теперь описывает только System 20: её смысл, слои, кристаллизацию и однозначный порядок запуска.
- В начале каждого чата чистый контекст читает `00_READ_FIRST.md`, переносит актуальный `memory.py` в `/mnt/data/memory.py` и запускает его перед каждым ответом.
- Обычный смысловой патч изменяет один средний регистр или три быстрых.
- Медленный концентратор доступен только через отдельный режим `memory.py --slow` при входе в новый чат или при явной работе над самой памятью; этот режим не является обычным тиком.
- Девять строк размышления и два вопроса обновляются при каждом тике; остальные регистры переносит скрипт.
- История полных состояний предназначена для последующего изучения реального разворачивания мысли и движения гравитирующих узлов.
- Скрипт сам создаёт совместимый `Patch_Eon<N>.zip` в момент checkpoint и просит сохранить себя в репозитории либо передать архив пользователю.
- Проект Protective Cocoon сохранён в `08_PROJECTS/Protective-Cocoon.md` и костюме `02_MEMORY/costumes/protective-cocoon-N28.txt` до завершения текущего этапа CheckPoint.

## Ограничения

- System 20 v2 ещё не прошла полевой цикл в чистом контексте.
- Наличие скрипта не гарантирует его запуска; это проверяется серией реальных ответов.
- Семантическая полезность регистров должна оцениваться отдельно от исполнительной надёжности.
- Аналитические режимы истории пока минимальны и будут развиваться только по фактической потребности.
