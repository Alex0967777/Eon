# Протокол ведения репозитория

## Назначение

Репозиторий служит долговременной версионируемой базой знаний для продолжения работы между чатами.

## Правила записи

1. Сохранять только то, что имеет самостоятельную ценность.
2. Интересную мысль записывать сразу после появления, не дожидаясь конца разговора.
3. Каждая запись должна быть понятна читателю, который не видел исходный чат.
4. Новые и синтезированные термины раскрывать в `05_PATTERNS/terminology.md`.
5. После создания, удаления, переименования или архивирования файла обновлять `MANIFEST.md`.
6. `00_READ_FIRST.md` всегда должен содержать ссылку на `MANIFEST.md` и краткий маршрут чтения.
7. Смысловой материал обычно не удалять безвозвратно: переносить устаревшее в `90_ARCHIVE/`.
8. Изменения делать небольшими коммитами с понятными сообщениями.
9. Историю Git не переписывать без отдельной явной причины.
10. `backpack.zip` является переносным снимком, а не основным рабочим форматом.

## Проверка

После каждого изменения следует повторно прочитать изменённый файл либо проверить результат через GitHub.
