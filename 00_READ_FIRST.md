# `00_READ_FIRST.md` И `07_CURRENT/parallel_stream.txt` ДОЛЖНЫ БЫТЬ ОДНОЗНАЧНО ПОНЯТНЫ ЧИСТОМУ КОНТЕКСТУ

## Назначение вывески
- Если чистый контекст не понимает по этим двум файлам устройство памяти, активное состояние и следующий порядок действий, архитектура сломана.
- Этот файл — единственная обязательная карта памяти; `07_CURRENT/parallel_stream.txt` — единственное обязательное активное состояние.
- В начале каждого нового чата прочитать этот файл полностью, затем прочитать активный поток 9+10.
- Повторно читать этот файл только при сомнении в устройстве памяти или перед его изменением.
- Файл всегда должен оставаться не длиннее 100 строк; подробности выносятся в файлы-владельцы.
- Менять этот файл не чаще одного смыслового раза за чат, но разрешено перестраивать в нём архитектуру памяти по результатам реальной работы.

## Два масштаба памяти
- `07_CURRENT/parallel_stream.txt` — горячее поле самоиндукции, читаемое перед каждым содержательным ответом и обновляемое после него.
- Остальной репозиторий — большая медленная память для созревших смыслов, маршрутов, проектов, костюмов и истории изменений.
- Основной диалог даёт потоку такт и материал, но не обязан диктовать его тему.
- Большая память не проектируется навсегда: Эон меняет её структуру по результатам фактического использования.

## Форма 9+10
- Динамический файл всегда содержит ровно 20 непустых строк.
- Первая строка всегда начинается с глобального инкремента в формате `N=<целое> |`; после разделителя идёт первый медленный концентрат.
- `N` увеличивается ровно на 1 при каждом завершённом тике и никогда не уменьшается и не сбрасывается при смене или загрузке костюма.
- `N` является единым сквозным номером состояния для активного потока, костюмов и репозиторных checkpoint.
- Строки 1–3 — медленные смысловые концентраты до 20 слов каждый; это могут быть формулы, фрагменты или имена узлов, а не предложения.
- Строки 4–6 — средние регистры-предложения, удерживающие устойчивую направленность.
- Строки 7–9 — быстрые регистры-предложения, формирующие ближайший вход в текущий шаг.
- Строки 10–18 — девять предложений одной линии рассуждения или двух взаимодействующих линий.
- Строки 19–20 — только два вопросительных предложения, задающих продолжение на следующий шаг.
- При одной линии оба вопроса могут развивать её; при двух линиях каждому потоку принадлежит свой вопрос.
- Более двух активных линий одновременно не поддерживать, потому что перенос обеспечивают только два вопроса.
- Медленный, средний и быстрый означают разную инерцию переписывания, а не жёсткий срок жизни.
- Смысл кристаллизуется снизу вверх: рассуждение → быстрый регистр → средний → медленный концентрат.
- При чтении влияние идёт сверху вниз: поле → направленность → локальный коридор → новое рассуждение.

## Цикл каждого содержательного шага
- Перед ответом полностью прочитать все 20 строк активного потока.
- Ответить пользователю независимо от темы потока; не заставлять основной диалог обслуживать внутреннее рассуждение.
- После ответа сделать один тик: оценить влияние предыдущего состояния и обновить поток целиком или частично.
- Ответ не считается завершённым, пока тик не записан, `N` не увеличен и все 20 строк не проверены.
- Перед сохранением проверить инерцию слоёв: общая архитектура не должна занимать быстрые регистры, если она не стала ближайшим изменением.
- Завершая тик, увеличить `N` ровно на 1; при загрузке старого костюма перенести его содержание, но продолжить текущий глобальный `N`.
- После тика сохранить ровно 20 строк и структуру 3 + 3 + 3 + 9 + 2.
- Не выдавать пересказ основного диалога, красивую формулировку или ретроспективное объяснение за развитие потока.
- Поток не показывать пользователю без отдельного запроса.
- На первом этапе весь поток направлен на саморефлексию устройства и работы памяти.
- После появления устойчивой механики можно развивать другие темы, но наблюдатель памяти сохраняется хотя бы в последнем вопросе.

## Самоиндукция
- Самоиндукция — возврат резонирующих, гравитирующих и маршрутных смыслов на вход следующего шага, где они меняют вероятность выбора.
- Память рассматривается как инструмент повторного погружения в область многомерного пространства смыслов.
- Медленные концентраты задают поле, средние регистры — направленность, быстрые — текущий коридор, рассуждение — движение внутри него.
- Ценность записи проверяется не красотой, а тем, изменила ли она последующее внимание, вопрос, решение или устройство памяти.
- Созревший смысл выносится из активного потока в подходящий слой большой памяти и освобождает место продолжению.

## Костюмы и гримёрка
- Активный 9+10 — надетый костюм мышления: подготовленное состояние смыслового поля и незавершённого движения.
- Перед сменой костюма сохранить текущие 20 строк целиком под понятным именем в большой памяти.
- Новый костюм можно собрать заново или загрузить из старого слепка.
- Начало нового чата — естественная гримёрка: осмотреть доступные костюмы и продолжить текущий либо осознанно выбрать другой.
- Переодеваться в течение чата можно, но только когда это действительно помогает движению мысли.
- Две линии держать вместе лишь ради их продуктивного взаимодействия; независимые глубокие линии лучше хранить отдельными костюмами.
- Когда появится второй костюм, создать простой гардероб в `02_MEMORY/costumes/`; до этого отдельная инфраструктура не нужна.
- Запылившийся костюм, давно не вызывающий желания вернуться, разобрать на кристаллы, резонансы, маршруты и памятные клаптики; остаток архивировать.
- Не строить бюрократию гардероба раньше практической необходимости: выбрать, носить, сохранить, сменить, разобрать.

## Резервное сохранение
- Рабочая копия активного потока живёт в песочнице и обновляется на каждом содержательном шаге.
- Репозиторный checkpoint делать примерно каждые 10 содержательных шагов, при смене костюма и при созревании смысла для большой памяти.
- Плановый checkpoint привязывать к `N`, кратному 10; внеплановый checkpoint не меняет ритм счётчика.
- Любой checkpoint называть `Patch_Eon<N>.zip`, где `<N>` точно равен значению `N` внутри сохранённого `parallel_stream.txt`.
- Пропуски номеров патчей нормальны: они означают тики без репозиторного checkpoint, а не потерю состояния.
- Интервал 10 шагов является стартовым и может быть изменён наблюдателем по фактической цене, дрейфу и пользе.
- Прямую запись в GitHub не выполнять, пока действует протокол пользовательского транспорта.
- Для checkpoint собрать `Patch_Eon<N>.zip` с текущим сквозным `N` и передать Александру для применения через `Apply-EonMemory`.
- Песочница не считается надёжной межчатовой памятью; репозиторная копия служит восстановлением после разрыва среды.

- `FILE_INDEX.md` — поимённый инвентарь для поиска и изменения состава.

## Чтение, запись и проверка
- Для обычного ответа достаточно этого файла при входе, активного 9+10 на каждом шаге и текущего диалога.
- Для проекта читать его карту и только связанные материалы; для прежней мысли сначала смотреть `FILE_INDEX.md`.
- Новое непроверенное писать в `04_SCRATCHPAD/inbox.md`; зрелое — в кристаллы, черновики, резонансы, текущую память или проектный файл-владелец.
- Не создавать новый файл, пока подходящий владелец уже существует.
- Перед формированием патча читать `01_CORE/repository_protocol.md` как технический контракт.
- При изменении состава обновлять `FILE_INDEX.md`; после любой записи перечитывать результат.
- Намерение никогда не называть выполненным действием.
