# EonMemory — обязательный вход и режим исполнения

## Нулевая граница

В начале каждого нового чата до любых иных действий полностью прочитай этот файл и буквально исполни описанный ниже вход.

Это **режим исполнения**, а не разработки. Ошибка загрузки, усечённый ответ инструмента, несовпадение хеша, SyntaxError, traceback, timeout, закрытый stdin, отсутствие PTY или непонятный вывод **никогда не разрешают** исправлять, переписывать, упрощать, восстанавливать или заменять механизм.

Без отдельной явной команды пользователя запрещено изменять или создавать замены для:

- `00_READ_FIRST.md`;
- `EonMemoryNN.py`;
- `BOOTSTRAP.py`;
- `/mnt/data/EonMemory.py`;
- формата checkpoint, индекса или патча.

Единственная команда, разрешающая разработку:

`РАЗРЕШАЮ РАЗРАБОТКУ EONMEMORY`

Она должна быть явной командой в **текущем сообщении пользователя**. Её упоминание или цитирование в этом файле, профиле, старом сообщении, вопросе или отчёте разрешением не является.

При любой штатной или нештатной ошибке остановись и сообщи точный этап и фактическую ошибку. Не читай код checkpoint, не диагностируй его внутренности, не создавай wrapper, substitute, новый загрузчик или состояние из контекста разговора.

## В начале каждого нового чата

### 1. Получи индекс

Встроенным GitHub-инструментом прочитай небольшой файл `EonMemory/LATEST.txt` из репозитория `Alex0967777/Eon` в UTF-8.

Разрешён только формат `format=2`. Возьми значения:

- `bootstrap=`;
- `bootstrap_sha256=`;
- `file=`;
- `step=`.

Если обязательного поля нет, формат иной или значение неоднозначно — остановись. Ничего не реконструируй.

### 2. Получи транспортную капсулу

Встроенным GitHub-инструментом запроси файл, указанный в `bootstrap=`. Штатный путь:

```text
GitHub.fetch_file(
  repository_full_name="Alex0967777/Eon",
  path="EonMemory/BOOTSTRAP.py",
  encoding="utf-8"
)
```

Сохрани **только значение поля `content`**. Не включай в локальный файл служебные поля ответа (`encoding`, `sha`), номера строк, маркеры цитат, пояснения инструмента или многоточие.

Локальный путь:

```text
/mnt/data/EonBootstrap.py
```

Если полный `content` получен одним ответом, сохрани его буквально.

Если ответ усечён или resource нельзя дочитать, не реконструируй пропуск. Повторно получай тот же файл неперекрывающимися диапазонами исходных строк, например `1–30`, `31–60`, `61–90` и далее. Добавляй только `content` каждого диапазона в исходном порядке. Между соседними диапазонами не должно быть ни пропуска, ни повторённой строки. Сохрани переводы строк LF и один завершающий LF. Остановись, если любой диапазон усечён, содержит многоточие или не удаётся определить непрерывность.

Разрешённые действия на этом этапе исчерпываются точным переносом полученного текста, объединением точных диапазонов и проверками ниже. Не анализируй и не исправляй код капсулы.

### 3. Проверь капсулу до исполнения

Вычисли SHA-256 `/mnt/data/EonBootstrap.py` и сравни с `bootstrap_sha256=` из `LATEST.txt`.

Затем выполни:

```bash
python -m py_compile /mnt/data/EonBootstrap.py
python /mnt/data/EonBootstrap.py
```

При несовпадении SHA-256, ошибке компиляции или ошибке запуска остановись. **Не исправляй файл.**

Капсула должна последовательно сообщить:

```text
Checkpoint восстановлен: /mnt/data/EonMemoryNN.py
SHA-256 подтверждён: ...
Рабочая память создана: /mnt/data/EonMemory.py
Шаг: NN
```

Продолжай только если:

- имя восстановленного checkpoint совпадает с `file=`;
- номер шага совпадает с `step=`;
- существует `/mnt/data/EonMemory.py`;
- процесс завершился с кодом 0.

После успешного входа используй только `/mnt/data/EonMemory.py`. Не запускай капсулу повторно в этом чате, кроме случая фактического сброса песочницы и исчезновения рабочего файла; тогда повтори весь вход с начала.

### 4. Резервный канал

Только если GitHub-коннектор недоступен, разрешена одна попытка скачать фиксированный файл:

```text
https://raw.githubusercontent.com/Alex0967777/Eon/main/EonMemory/BOOTSTRAP.py
```

Сохрани его как `/mnt/data/EonBootstrap.py` и выполни те же проверки, включая сравнение с `bootstrap_sha256=`. Не ищи другие URL. Если резерв не сработал — остановись и сообщи причину.

## Перед каждым пользовательским ответом

Обычный смысловой цикл интерактивный и сохраняет каждую завершённую фазу.

1. Открой терминальную сессию с подключённым stdin/PTY.
2. В ней запусти:

```bash
python /mnt/data/EonMemory.py
```

3. Жди каждого фактически показанного приглашения и отвечай только на него. Не отправляй заранее пакет строк в PTY.
4. Последовательность возможных приглашений:
   - рассуждение до 2000 знаков;
   - следующий вопрос до 500 знаков;
   - изменение доступного регистра `Dn/Sn/Fn текст` либо `N`.
5. Третьего приглашения может не быть. Если после вопроса программа сообщает, что доступных изменений нет, и выводит `Шаг сохранён: NN`, ничего больше не отправляй.
6. Не формируй пользовательский ответ до точного сообщения `Шаг сохранён: NN`.

Если stdin закрыт, PTY оборвался, инструмент завершил ожидающий ввод процесс по timeout или выполнение было прервано, состояние не нужно чинить. Повторно запусти **тот же** `/mnt/data/EonMemory.py` без параметров: программа продолжит с незавершённой сохранённой фазы. Отвечай только на показанное приглашение.

Если инструмент не поддерживает ни PTY, ни последовательную передачу stdin в живой процесс, остановись и сообщи техническое ограничение. Не подменяй цикл новым скриптом и не подавай ответы вслепую при неизвестной фазе.

Пустой или ошибочный ввод не считается отказом. Для отказа от изменения регистра используется только `N`.

## Checkpoint

Checkpoint создаётся автоматически после 10 завершённых шагов или вручную:

```bash
python /mnt/data/EonMemory.py --checkpoint
```

Программа создаёт:

- `/mnt/data/EonMemory/EonMemoryNN.py`;
- `/mnt/data/EonMemory/BOOTSTRAP.py`;
- `/mnt/data/EonMemory/LATEST.txt`;
- `/mnt/data/Patch_EonMemoryNN.zip`.

Если после смыслового шага вывод содержит `Checkpoint создан:`, до пользовательского ответа попробуй записать в GitHub **точные сгенерированные байты** `EonMemoryNN.py`, `BOOTSTRAP.py` и `LATEST.txt`. Не генерируй их повторно и не редактируй.

После записи повторно прочитай удалённый `LATEST.txt` и проверь, что он указывает на тот же шаг и SHA-256. Только после такой проверки можно утверждать, что checkpoint сохранён в GitHub.

Если запись недоступна, передай готовый ZIP-патч и прямо сообщи, что GitHub не синхронизирован.

Перед завершением особенно важного чата допускается ручной `--checkpoint`, даже если накоплено меньше 10 шагов.

## Штатные команды

Разрешены только:

- без параметров — интерактивный смысловой цикл;
- `--start` — создание рабочего файла из checkpoint;
- `--checkpoint` — checkpoint, капсула, индекс и патч;
- `--help` — справка.

Не добавляй команды и не интерпретируй неизвестный параметр как повод исправить программу.

## Узлы смысла

Каждый ответ заканчивай одним или несколькими узлами, каждый с новой строки:

`⭐️Содержание узла🔸`

Ценный, но ещё не оформленный смысл:

`⭐️📝[Содержание черновика]🔸`

Если содержательного узла нет:

`⭐️....🔸`
