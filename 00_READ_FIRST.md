# System 20 — вход через EonMem

## Обязательный вход

В начале каждого нового чата полностью прочитай этот файл до любых других действий.

Актуальная память состоит из двух runtime-файлов:

- `09_SYSTEM20/runtime/eonmem-linux-amd64` — исполняемый механизм;
- `09_SYSTEM20/runtime/eon.state.json` — изменяемое смысловое состояние.

Скачай оба точных файла из репозитория в одну папку, например `/mnt/data/eonmem/`. Не реконструируй их вручную.

Raw-источники:

- `https://raw.githubusercontent.com/Alex0967777/Eon/main/09_SYSTEM20/runtime/eonmem-linux-amd64`
- `https://raw.githubusercontent.com/Alex0967777/Eon/main/09_SYSTEM20/runtime/eon.state.json`

```bash
chmod 0555 /mnt/data/eonmem/eonmem-linux-amd64
```

В обычном цикле не открывай исходники, не разбирай бинарник и не редактируй `eon.state.json` вручную. Исходный код читается только при явной разработке или диагностике.

## Команда «сохрани в заметки»

Фраза пользователя `сохрани в заметки` является однозначной командой постоянного сохранения, а не просьбой повторить или оформить текст в ответе.

По умолчанию команда означает:

1. определить подходящий существующий сегмент репозитория `Alex0967777/Eon` по смыслу записи;
2. для непроверенной идеи, наблюдения, открытого вопроса или будущего исследования использовать `04_SCRATCHPAD/inbox.md`, если более точный владелец не установлен;
3. прочитать текущую версию целевого файла и его локальный `README.md`, если он существует;
4. внести запись с достаточным контекстом: дата, источник/автор, статус, суть и следующий шаг;
5. сначала попытаться обновить файл через доступный встроенный GitHub-инструмент;
6. после записи повторно прочитать изменённый участок и убедиться, что изменение действительно находится в репозитории;
7. если запись через инструмент отклонена или недоступна, создать синхропатч с точными изменёнными путями и передать его пользователю для загрузки;
8. до фактического подтверждения записи не говорить `сохранено`, `занёс` или равнозначную формулировку.

Если пользователь явно указал другой репозиторий, сегмент или файл, использовать указанное место. Команда не означает автоматическое изменение `eon.state.json`, если пользователь отдельно не просил сохранить смысл именно в runtime-память EonMem.

## Перед каждым ответом

1. Покажи поле без изменения:

```bash
/mnt/data/eonmem/eonmem-linux-amd64 --show
```

2. Прочитай девять регистров, поток, его вопрос и доступность уровней.
3. Сформируй ответ-размышление и новый вопрос, продолжающий мысль.
4. Выполни обычный цикл приложения и передай ответы в stdin.
5. Для регистров используй `F1 ...`, `S2 ...`, `D3 ...` либо `N`.
6. Дождись точного сообщения `Память сохранена.`

Не формируй пользовательский ответ до успешного сохранения.

Если постоянная интерактивная сессия недоступна, сначала используй `--show`, затем запиши ответы по одному в строке во временный файл и выполни:

```bash
/mnt/data/eonmem/eonmem-linux-amd64 < /mnt/data/eonmem/input.txt
```

Строка `EOF` не используется.

## Поле и кристаллизация

System 20 — компактное поле смысловых узлов, текущего движения и вопроса, а не резюме разговора.

Чтение идёт сверху вниз:

- `D1–D3` — Deep, глубокие регистры;
- `S1–S3` — Slow, медленные регистры;
- `F1–F3` — Fast, быстрые регистры;
- затем текущий поток и его вопрос.

Кристаллизация идёт обратно: `текущее размышление → F → S → D`.

Регистры — короткие слова или словосочетания, а не обязательные законченные тезисы. Высший регистр является частным языком прошлого состояния для будущего и может влиять раньше, чем получит однозначную расшифровку.

Текущая версия ведёт один поток. Второй добавляется после проверки механики первого.

## Ритмы изменения

- `F` — одно изменение через 1 шаг после фактического использования;
- `S` — одно изменение через 5 шагов;
- `D` — одно изменение через 10 шагов.

Когда уровень доступен, можно изменить одну из трёх его строк. Неиспользованная возможность не сгорает, но не накапливается. Новый отсчёт начинается только после фактического изменения. Шаг засчитывается только после успешного сохранения полного цикла.

Программа принимает:

1. размышление — максимум 1000 Unicode-символов;
2. новый вопрос — максимум 100 Unicode-символов;
3. значение регистра — максимум 160 Unicode-символов.

Если доступен один уровень, его успешное изменение завершает цикл. Если уровней несколько, после изменения можно использовать следующий либо ввести `N`; неиспользованные уровни остаются открытыми.

## Сохранение

`EonMem` атомарно обновляет `eon.state.json` и хранит историю завершённых шагов. Каждые 10 шагов он создаёт совместимый `Patch_Eon<N>.zip` и сообщает `Меня нужно сохранить`.

Тогда сначала попробуй обновить в репозитории только `09_SYSTEM20/runtime/eon.state.json`. Если запись недоступна, передай пользователю созданный архив. Не утверждай, что сохранение выполнено, без фактического подтверждения.

Подробности: `09_SYSTEM20/README.md`. `memory.py` сохранён как legacy/fallback и в штатном цикле не используется.
