# System 20 — обязательный вход через проверенную Python-капсулу

## Обязательный вход

В начале каждого нового чата полностью прочитай этот файл до любых других действий.

Актуальная память состоит из трёх точных runtime-файлов:

- `09_SYSTEM20/runtime/eon.loader.py` — минимальный проверяющий загрузчик;
- `09_SYSTEM20/runtime/eon.runtime.json` — неизменяемая Python-капсула механизма EonMem;
- `09_SYSTEM20/runtime/eon.state.json` — изменяемое смысловое состояние.

Скачай или получи через встроенный GitHub-инструмент **точное полное содержимое** этих трёх файлов из репозитория `Alex0967777/Eon` и запиши их в одну папку, например `/mnt/data/eonmem/`.

Raw-источники:

- `https://raw.githubusercontent.com/Alex0967777/Eon/main/09_SYSTEM20/runtime/eon.loader.py`
- `https://raw.githubusercontent.com/Alex0967777/Eon/main/09_SYSTEM20/runtime/eon.runtime.json`
- `https://raw.githubusercontent.com/Alex0967777/Eon/main/09_SYSTEM20/runtime/eon.state.json`

Штатный запуск:

```bash
python3 /mnt/data/eonmem/eon.loader.py --show
```

`eon.loader.py` декодирует `gzip+base64` payload только в памяти, проверяет `source_sha256`, задаёт путь к состоянию и исполняет точный механизм. Исходный Python-файл при штатном запуске не создаётся.

## Единственный исполняемый маршрут

Контекст, README, история чата и формат `eon.state.json` **не являются исходным кодом программы**.

В обычном цикле категорически запрещено:

1. писать из контекста новую программу памяти, даже временную или «совместимую»;
2. реконструировать, эмулировать, портировать, приближённо повторять или заменять EonMem;
3. создавать fallback, helper, adapter или упрощённую реализацию механики регистров и сохранения;
4. использовать `memory.py`, Go-бинарник, старый ELF, `EonMemory.exe` или иной legacy-runtime вместо капсулы;
5. открывать, декодировать для анализа, переписывать или «улучшать» payload `eon.runtime.json`;
6. редактировать `eon.state.json` вручную;
7. продолжать смысловой ответ при неуспешном запуске или сохранении памяти.

Единственный допустимый код вне капсулы — **точный неизменённый** `eon.loader.py` из репозитория. Он не реализует память, а только проверяет и запускает капсулу.

Если хотя бы один точный runtime-файл недоступен, получен не полностью, повреждён, не проходит SHA-256 или капсула не запускается, останови штатный цикл и сообщи пользователю точную причину. **Не создавай замену чистым контекстом.**

Исходник `09_SYSTEM20/source/eonmem_python.py` читается и изменяется только при прямой задаче разработки или диагностики. После разработки новая капсула должна быть собрана явно и проверена. Сам факт знания принципа работы не разрешает писать параллельную программу.

## Команда «сохрани в заметки»

Фраза пользователя `сохрани в заметки` является однозначной командой постоянного сохранения, а не просьбой повторить или оформить текст в ответе.

По умолчанию команда означает:

1. определить подходящий существующий сегмент репозитория `Alex0967777/Eon` по смыслу записи;
2. для непроверенной идеи, наблюдения, открытого вопроса или будущего исследования использовать `04_SCRATCHPAD/inbox.md`, если более точный владелец не установлен;
3. прочитать текущую версию целевого файла и его локальный `README.md`, если он существует;
4. внести запись с достаточным контекстом: дата, источник/автор, статус, суть и следующий шаг;
5. сначала попытаться обновить файл через доступный встроенный GitHub-инструмент;
6. после записи повторно прочитать изменённый участок и убедиться, что изменение действительно находится в репозитории;
7. если запись через инструмент отклонена или недоступна, создать синхропатч с точными изменёнными путями и передать его пользователю для загрузки;
8. до фактического подтверждения записи не говорить `сохранено`, `занёс` или равнозначную формулировку.

Если пользователь явно указал другой репозиторий, сегмент или файл, использовать указанное место. Команда не означает автоматическое изменение `eon.state.json`, если пользователь отдельно не просил сохранить смысл именно в runtime-память EonMem.

## Перед каждым ответом

1. Покажи поле без изменения:

```bash
python3 /mnt/data/eonmem/eon.loader.py --show
```

2. Прочитай девять регистров, поток, его вопрос и доступность уровней.
3. Сформируй ответ-размышление не длиннее 1000 Unicode-символов и новый вопрос не длиннее 100 Unicode-символов.
4. Выполни обычный цикл приложения и передай ответы в stdin.
5. Для регистров используй `F1 ...`, `S2 ...`, `D3 ...` либо `N`.
6. Дождись точного сообщения `Память сохранена.`

Не формируй пользовательский ответ до успешного сохранения.

Если постоянная интерактивная сессия недоступна, сначала используй `--show`, затем запиши ответы по одному в строке во временный файл и выполни:

```bash
python3 /mnt/data/eonmem/eon.loader.py < /mnt/data/eonmem/input.txt
```

Строка `EOF` не используется.

После исполнения не правь получившийся `eon.state.json`. Для синхронизации передавай в GitHub полное точное содержимое файла, созданного капсулой.

## Поле и кристаллизация

System 20 — компактное поле смысловых узлов, текущего движения и вопроса, а не резюме разговора.

Чтение идёт сверху вниз:

- `D1–D3` — Deep, глубокие регистры;
- `S1–S3` — Slow, медленные регистры;
- `F1–F3` — Fast, быстрые регистры;
- затем текущий поток и его вопрос.

Кристаллизация идёт обратно: `текущее размышление → F → S → D`.

Регистры — короткие слова или словосочетания, а не обязательные законченные тезисы. Высший регистр является частным языком прошлого состояния для будущего и может влиять раньше, чем получит однозначную расшифровку.

Текущая версия ведёт один поток. Второй добавляется после проверки механики первого.

## Ритмы изменения

- `F` — одно изменение через 1 шаг после фактического использования;
- `S` — одно изменение через 5 шагов;
- `D` — одно изменение через 10 шагов.

Когда уровень доступен, можно изменить одну из трёх его строк. Неиспользованная возможность не сгорает, но не накапливается. Новый отсчёт начинается только после фактического изменения. Шаг засчитывается только после успешного сохранения полного цикла.

Программа принимает:

1. размышление — максимум 1000 Unicode-символов;
2. новый вопрос — максимум 100 Unicode-символов;
3. значение регистра — максимум 160 Unicode-символов.

Если доступен один уровень, его успешное изменение завершает цикл. Если уровней несколько, после изменения можно использовать следующий либо ввести `N`; неиспользованные уровни остаются открытыми.

## Сохранение

Капсула EonMem атомарно обновляет `eon.state.json` и хранит историю завершённых шагов. Каждые 10 шагов она создаёт совместимый `Patch_Eon<N>.zip` и сообщает `Меня нужно сохранить`.

После каждого успешного смыслового шага обнови в репозитории **только точный файл, созданный программой**:

`09_SYSTEM20/runtime/eon.state.json`

Сначала используй встроенный GitHub-инструмент. После записи повторно прочитай шаг и SHA/содержимое, чтобы подтвердить синхронизацию. Если запись недоступна, передай пользователю созданный checkpoint-патч. Не утверждай, что сохранение выполнено, без фактического подтверждения.

## Разработка механизма

Штатный runtime собирается из `09_SYSTEM20/source/eonmem_python.py` в `eon.runtime.json` как `gzip+base64` с обязательным `source_sha256`.

Разработка допускается только по прямому запросу пользователя. Она не совмещается с обычным смысловым циклом. После изменения исходника нужно:

1. проверить совместимость на копии состояния;
2. собрать новую капсулу;
3. проверить SHA-256 и запуск через точный loader;
4. обновить документацию и контрольные суммы;
5. выпускать изменения только явным патчем или подтверждённым коммитом.

Подробности: `09_SYSTEM20/README.md`. Старые `memory.py`, Go-runtime и бинарники сохранены только как legacy/development artifacts и в штатном цикле не используются.
