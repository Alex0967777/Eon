# EonMem — исполняемая System 20 в Python-капсуле

## Назначение

`EonMem` отделяет жёсткую механику памяти от смыслового поля. В штатном маршруте модель не читает и не редактирует исходный код, не правит JSON вручную и не создаёт параллельных реализаций из контекста. Она запускает точный loader, читает поле, отвечает на приглашения и ждёт подтверждения сохранения.

Текущая логика совместима с форматом состояния v1 и поведением EonMem 0.1.1.

## Структура

- `runtime/eon.loader.py` — минимальный загрузчик: проверяет формат, декодирует капсулу, сверяет SHA-256 и исполняет payload в памяти.
- `runtime/eon.runtime.json` — неизменяемая `gzip+base64` Python-капсула механизма.
- `runtime/eon.state.json` — изменяемое состояние; должно лежать рядом с loader и капсулой.
- `runtime/SHA256SUMS` — контрольные суммы runtime-файлов.
- `source/eonmem_python.py` — канонический открытый исходник только для явной разработки и диагностики.
- прежние Go/Windows runtime-файлы — legacy/development artifacts, не штатный маршрут нового чата.

## Инвариант единственного механизма

В обычном цикле нельзя писать, реконструировать, эмулировать, портировать или запускать альтернативную программу памяти. README и JSON описывают протокол, но не дают разрешения реализовать его заново.

Если точный loader, капсула или состояние недоступны либо не проходят проверку, цикл останавливается с ошибкой. Создание временного fallback запрещено.

## Один смысловой шаг

Программа показывает:

1. девять регистров в порядке `D → S → F`;
2. доступность изменения каждого уровня;
3. текущее размышление потока 1;
4. текущий вопрос потока 1.

Затем принимает:

1. ответ-размышление до 1000 Unicode-символов;
2. новый вопрос, продолжающий мысль, до 100 Unicode-символов;
3. допустимые изменения регистров строками вида `F1 новое значение`, `S2 новое значение`, `D3 новое значение`;
4. `N`, если оставшиеся доступные возможности не используются.

Значение регистра ограничено 160 Unicode-символами.

## Ритмы регистров

- `F` — Fast: одно изменение уровня после каждого фактического использования через 1 шаг.
- `S` — Slow: одно изменение уровня через 5 шагов после фактического использования.
- `D` — Deep: одно изменение уровня через 10 шагов после фактического использования.

Разрешение не сгорает, если его не использовать, но и не накапливается. Когда уровень доступен, можно изменить ровно одну из трёх его строк. Отсчёт нового интервала начинается только после фактического изменения уровня.

Шаг увеличивается только после успешной атомарной записи состояния.

## Запуск

```bash
python3 ./eon.loader.py
```

### Просмотр без изменения

```bash
python3 ./eon.loader.py --show
```

`--show` печатает регистры, доступность и поток, после чего завершается без изменения состояния.

### Надёжный пакетный ввод для песочницы

Сначала выполнить `--show`, затем записать ответы по одному в строке во временный файл и передать его в stdin:

```bash
python3 ./eon.loader.py < /mnt/data/eonmem/input.txt
```

Порядок строк определяется показанной доступностью:

1. размышление;
2. новый вопрос;
3. по одной строке на каждый используемый доступный уровень;
4. `N`, если после изменений остаются неиспользованные доступные уровни.

Специальный маркер `EOF` не используется.

## Проверка капсулы

Loader принимает только:

- `format = EonRuntimeCapsule`;
- `version = 1`;
- `engine = python3`;
- `codec = gzip+base64`.

После декодирования вычисляется SHA-256 исходника и сравнивается с `source_sha256`. При любом несовпадении запуск запрещается. Loader не записывает исходник на диск.

## Состояние и checkpoint

Состояние записывается атомарно рядом с loader в `eon.state.json`. Программа хранит полную историю завершённых шагов.

Каждые 10 завершённых шагов приложение создаёт рядом с runtime архив `Patch_Eon<N>.zip`, совместимый с `Apply-EonMemory`. Внутри находится актуальный файл `09_SYSTEM20/runtime/eon.state.json`.

После сообщения `Меня нужно сохранить` сначала следует попытаться обновить `09_SYSTEM20/runtime/eon.state.json` в репозитории. Если это недоступно, готовый архив передаётся пользователю.

## Разработка

Исходник открывается только по прямой задаче разработки или диагностики. Изменение механизма выполняется вне обычного смыслового цикла:

1. изменить `source/eonmem_python.py`;
2. проверить поведение на копии состояния;
3. детерминированно сжать исходник `gzip` с `mtime=0`;
4. записать Base64 в `runtime/eon.runtime.json` и обновить `source_sha256`;
5. проверить запуск через неизменённый `runtime/eon.loader.py`;
6. обновить `runtime/SHA256SUMS` и документацию;
7. выпустить явный патч.

Нельзя использовать знание протокола для создания отдельной «совместимой» программы.
